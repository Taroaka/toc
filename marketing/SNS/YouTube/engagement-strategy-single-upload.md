# 単発公開時のエンゲージメント戦略

最終更新: 2026-04-27
対象: `にわかのAI` の初回通常動画 `浦島太郎`  
目的: 視聴回数を最大化しつつ、次の視聴に繋がるコメントを増やす

---

## 1. 前提

- チャンネルは新規で、既存の視聴者はいない
- 公開物は `長尺1本 + サムネイル` が中心
- いま欲しいのは `雑な反応` ではなく、`動画内容に触れたコメント`
- コメントは飾りではなく、アルゴリズムへの追加シグナルと次回改善の材料
- CRM方針は `community-crm-strategy.md` を優先し、視聴者を `AIを一緒に育てる初期メンバー` として扱う

この前提では、`たくさんコメントしてください` よりも、`何について話してほしいか` を先に決めるほうが強い。

---

## 2. Pinned Comment Strategy

### 方針

- 自分のコメントを公開直後に pin する
- pin する文は `1. 人間の制作者がAIを育てている` `2. 未完成だが改善していく` `3. 1つだけ答えやすい質問` で作る
- 最初から外部リンクや大量の CTA は置かない
- コメントが育ってきたら、価値の高い視聴者コメントを pin に差し替える

### ねらい

- 視聴者が最初に読む文をコントロールする
- 何を話せばいいかを明示して、コメントの質を上げる
- 作品外の話題に逃がさず、動画内容への反応を増やす

### pin 文の設計ルール

- 質問は 1 つ
- Yes/No で終わる質問は避ける
- 感想だけを求めない
- `どこが印象に残ったか` `どの場面が怖かったか` `どの解釈が近いか` のように、具体的に答えられる問いにする

---

## 3. Description CTA Strategy

### 方針

説明欄の CTA は `1本だけ` に絞る。

- 最初の 2 行で `何の動画か` と `何を聞きたいか` を書く
- CTA は `コメント` を主軸にする
- 登録や高評価は補助に留める
- まだ次の動画がない段階では、外部リンクや導線を増やしすぎない

### 実務の型

1. 1行目で動画の要点を書く
2. 2行目でコメントしてほしい観点を示す
3. 最後に軽く登録を添える

### 例

```text
日本の民話「浦島太郎」を、AI映像で再解釈した1本です。
一番印象に残った場面、または解釈が気になった点をコメントで教えてください。
次の作品も見たい方はチャンネル登録で追ってもらえると助かります。
```

## 4. First Reply / Moderation Strategy

### 公開直後 24 時間

- `Newest` 並びでコメントを確認する
- 最初の有用コメントにはできるだけ早く返信する
- 良いコメントには heart を付ける
- 返信は短くてもよいが、必ず `追加情報` か `次の問い` を返す

### 返信の優先順位

1. 作品内容に触れているコメント
2. 解釈や考察があるコメント
3. 次の動画に繋がるコメント
4. 単純な称賛コメント
5. スパムや無関係コメント

### モデレーション

- `Hold potentially inappropriate comments for review` を前提にする
- URL を含むコメントは警戒する
- 連投の定型文、無関係な宣伝、露骨な誘導は早めに整理する
- 荒らしに反応しない

### 返信の型

- `見てくれてありがとう` だけで終わらせない
- 視聴者の言及ポイントを1つ拾って返す
- 可能なら軽い追質問を返して会話を伸ばす

例:

- `竜宮城の静けさを強く出したかったので、そこを拾ってもらえて嬉しいです。どのカットが一番印象に残りましたか?`

## 5. Pin に使える日本語例

### 例1: CRM標準型

```text
ここまで見てくれてありがとうございます。正直、まだ未完成なところも多くて、公開するのは少し恥ずかしいです。

でも、AIに任せきりではなく、人間の自分が「どんな物語として届くか」を見ながら、1本ずつ育てていきたいと思っています。

この浦島太郎で、いちばん印象に残った場面、または「ここはもっと見たかった」という部分を1つだけ教えてください。次の作品づくりに反映します。
```

### 例2: 解釈を拾う型

`一番印象に残ったのは、浦島太郎のどの場面でしたか。玉手箱の前と後で、見え方が変わった場面があれば知りたいです。`

### 例3: シーン特定型

`竜宮城の中で、一番不気味だった / 美しかった場面を1つ教えてください。場面名でも印象でも大丈夫です。`

### 例4: 視点切り替え型

`もしこの物語を浦島太郎ではなく乙姫側から見るなら、どこが一番気になりますか。`

### 例5: 次回改善に繋げる型

`映像、音、語りの中で、次回もっと強くしてほしい要素があれば教えてください。`

### 例6: 作品の感情を聞く型

`この動画でいちばん残った感情を1つだけ挙げるなら、驚き・切なさ・怖さのどれに近いですか。`

## 6. Useful Comments を増やす質問の型

### 効きやすい問い

- 二択や三択で答えられる問い
- 場面を特定できる問い
- 解釈の違いを引き出す問い
- 次回の制作判断に使える問い

### 具体例

- `玉手箱を開ける前後で、どちらの演出が強かったですか`
- `怖かったのは竜宮城そのものか、帰還後の変化か`
- `乙姫の印象は、優しさと不気味さのどちらに寄りましたか`
- `次に見たい日本の物語があれば、1つだけ挙げてください`

### 伸びやすい理由

- 視聴者が `何を書けばいいか` を迷わない
- 作品への理解や解釈がコメントとして残る
- 他人のコメントを読んでも面白いので、スレッドが伸びやすい

### 避ける問い

- `感想ください`
- `高評価お願いします`
- `コメント待ってます`
- `みんなはどう思う?`

これらは抽象的で、反応の質がぶれやすい。

---

## 7. 1本公開のときの実行順

1. 公開直後に pinned comment を入れる
2. 説明欄の冒頭 2 行を確認する
3. 最初のコメントに即返信する
4. 24 時間は `Newest` で追う
5. 反応が良い話題に絞って返信を続ける
6. 48 時間後に、pin を視聴者コメントへ差し替えるか判断する
