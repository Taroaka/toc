## 2. 最初に決めるブランドの核

### 2-1. 誰に向けるか

`にわかのAI` は、最初から全員向けにしない。
まずは次の2層に絞る。

| 層 | 期待する反応 | 役割 |
|----|-------------|------|
| 生成AI・AI動画・自動化に関心がある視聴者 | 仕組みに反応する、登録する、制作過程を追う | 初期流入の母体 |
| 民話・神話・映像表現が好きな視聴者 | 動画を最後まで見る、保存する、次の物語を待つ | チャンネル成長の母体 |
| AIエージェントによる制作システムに関心がある見込み顧客 | 問い合わせる、リスト登録する | 収益化の母体 |

### 2-2. 何者として覚えられるか

`にわかのAI` の立ち位置は `AI動画チャンネル` だけでは弱い。
次の一文で説明できる状態にする。

> にわかのAIは、生成AIで世界中のナラティブを再現し、語り継ぐチャンネル。物語名を送るだけで、台本設計から動画生成までAIエージェントが自動で組み立てる仕組みも見せる。

補足:

- 初期の入口は `AI好き` に置く
- 継続視聴の理由は `物語体験` に置く
- 差別化は `サービス手操作ではなく、プログラム化された全自動生成` に置く

### 2-3. 何を約束するか

ブランドは雰囲気より `視聴者への約束` で固める。

- 世界中のナラティブを、安っぽい切り抜きではなく `作品` として見せる
- AI生成であることを隠さず、全自動生成の仕組みも価値に変える
- 世界観は幻想的でも、チャンネルの説明は分かりやすくする

## 3. ブランディング設計

### 3-1. ブランド軸

次の3軸を固定する。

| 軸 | 推奨方針 |
|----|---------|
| 世界観 | 神秘、余韻、古代感、静かな没入 |
| 見え方 | 作品感を出すが、難解にしすぎない |
| 語り口 | 気取らず、しかし軽薄にしない。人間の制作者がAIを育てている温度感を出す |

### 3-2. 避けるべきブランドのズレ

- AIツール自慢だけのチャンネルに見えること
- 雑学解説チャンネルに見えること
- 子ども向け昔話読み聞かせに誤認されること
- 高級感を狙いすぎて入口が分かりにくくなること
- 完全自動や万能AIだけを強調して、人間の制作者の判断や改善過程が消えること
- 金髪ホスト風の見た目を、過剰な夜職ノリや煽りに寄せすぎること

### 3-3. ブランドメッセージの型

プロフィールや説明文の核は次の3点で作る。

1. 何を作っているか  
2. なぜそれを作るか  
3. どこに導線があるか

短い型:

```text
生成AIで世界中のナラティブを再現し、語り継ぐチャンネルです。
作品公開と全自動生成の裏側を通じて、物語とAIエージェント制作の両方を届けます。
制作やライセンスの相談は説明欄から。
```

## 4. チャンネル名の設計

### 4-1. 命名の判断基準

- 日本語話者に覚えやすい
- 将来の英語展開でも破綻しない
- 物語だけでなく制作システム導線にも伸ばせる

### 4-2. 推奨の考え方

最初は `ブランド名主軸 + 補助説明` が扱いやすい。

例:

- `にわかのAI`
- `物語アトラス`
- `物語綺譚`

実務ルール:

- `display name` と `@handle` は同じ語幹で揃える
- `@handle` は ASCII 寄りで、他サービスにも展開しやすいものを優先する
- 名前変更も handle 変更も頻繁にやる前提にしない

### 4-3. 避けたほうがいい命名

- 民話タイトルそのものをチャンネル名にする
- 流行語に寄せすぎた AI 感の強い名前
- 将来の事業導線と切れてしまう匿名的な名前

## 5. ビジュアルアイデンティティ

### 5-1. アイコン

役割は `小さくても認識されること`。
情報を詰め込まない。

推奨:

- 金髪ホスト風の顔アップを使う場合は、親しみやすさと未完成AIのキャラクター性を出す
- 肌色は日本人として自然な明るめの肌色にする
- 高コントラスト
- 円形トリミング前提
- 小さく表示されても目と髪型で認識できること

候補:

- 金髪ホスト風のオリジナルキャラクター顔アップ
- `N` や `NA` のモノグラム
- 玉手箱、月、鳥居、波紋など民話連想モチーフを背景で抽象化

### 5-2. バナー

役割は `一目で何のチャンネルか伝えること`。

実装前提:

- 推奨サイズは `2560 x 1440`
- 最小は `2048 x 1152`
- 文字やロゴは安全領域 `1235 x 338` に収める

バナーに入れる要素:

- チャンネル名
- 一文での価値説明
- 世界観を示すキービジュアル

バナーの一文例:

- `民話・神話・伝承を、映像でたどる。`
- `古い物語を、新しい手法で。`
- `作品公開と制作過程の公開。`

### 5-3. サムネとバナーの関係

バナーだけ荘厳で、サムネだけ煽り系になるとブランドが壊れる。
次の一貫性を守る。

- 色温度
- フォントのトーン
- 感情の強さ
- AI映像の見せ方

## 6. チャンネル説明文の設計

### 6-1. 説明文で満たすべき要件

- 何のチャンネルかが2行で分かる
- `AI` と `世界中のナラティブ` の両方が入る
- 制作や相談導線がある
- 誇張せず、期待値を正しく置く

### 6-2. 初期版テンプレート

```text
にわかのAIは、生成AIで世界中のナラティブを再現し、語り継ぐチャンネルです。
人間の制作者がAIエージェントを育てながら、物語名から台本設計、映像、ナレーション、動画生成までを組み立てています。
作品を楽しみたい方、改善提案を送りたい方、AI動画制作システムに興味がある方はこちら。

[制作ノート・改善提案・相談リンク]
```

補足:

- description は `何のチャンネルか / 何を扱うか / 次の行動` の3文で十分
- 初期リンクは増やしすぎず、まずは集約先1本を強調する
- YouTube 上のチャンネルリンクは最大 14 件だが、初期運用は 1〜3 件でよい

## 7. 初期設定で決めるべき項目

### 7-1. アカウント基礎

- チャンネル名
- `@handle`
- アイコン
- バナー
- 説明文
- 外部リンク
- 連絡先
- 運用主体を `Brand Account / channel permissions` にするか

### 7-2. YouTube 上の見え方

- チャンネルトップのレイアウト
- 注目動画の設定
- 再生リストの並び順
- Featured sections
- チャンネルキーワード

実務メモ:

- `Home tab` は必ず有効化する
- カスタム section は最大 12 まで置けるが、初期は 3〜5 で十分
- 初期状態のままだと generic に見えやすいので、手動で整える

### 7-3. 運用ルール

- コメント返信のトーン
- 概要欄テンプレート
- タイトル命名ルール
- サムネの最低基準
- 投稿曜日と時間

## 9. 再生リストの初期設計

最初から再生リストを分けておくと、チャンネルの意図が伝わりやすい。

推奨:

- `Start Here`
- `AIでつくる民話・神話`
- `制作の裏側`
- `Shorts`
- 将来用: `World Narrative` または `English Subtitled`

ルール:

- 日付順の箱ではなく、`視聴者が次に何を見るか` で名前を付ける
- playlist は public にして、channel top に出す前提で作る
- 初期は 2〜4 本の柱で十分

## 10. コメントとコミュニティの初期方針

### 10-1. コメント返信

- 丁寧だが説明過多にしない
- AI制作への質問には歓迎姿勢を見せる
- 攻撃的なコメントに感情で返さない

### 10-2. チャンネルの人格

おすすめは `職人 + 案内人`。

- 偉そうにしない
- 友達ノリに寄せすぎない
- 技術説明はできるが、作品の空気を壊さない

補足:

- 視聴者への第一約束は `作品体験`
- 制作透明性は `信頼の証拠` として後段で効かせる

---

## 11. 初期によくある失敗

- チャンネル名と動画タイトルのトーンが合っていない
- バナーを作っただけで、トップページ導線を整えていない
- 説明文に問い合わせ導線がない
- AIチャンネルなのか民話チャンネルなのか、初見で分からない
- 最初の1本だけ豪華で、2本目以降の運用ルールがない

## 12. にわかのAI における推奨初期形

迷うなら次で始める。

- チャンネル名: `にわかのAI`
- handle: `@niwakanoai`
- ブランド説明: `生成AIで世界中のナラティブを再現し、語り継ぐ。物語名から動画完成までをAIエージェントが自動で組み立てる`
- 再生リスト: `まず見る3本 / AIでつくる民話・神話 / 制作の裏側`
- トーン: `神秘的だが閉じない。作品感はあるが説明は明快`
- 導線: `動画 -> 概要欄 -> LP / 問い合わせ`
- launch 時の page 設計: `trailer / featured video / Start Here / Core series`
