# 浦島太郎 通常投稿プレイブック

最終更新: 2026-04-12  
対象: `にわかのAI` の初回通常動画 `浦島太郎`  
前提: あるのは `動画` と `サムネイル` だけ。Shorts なし。広告なし。新規チャンネル。  
目的: 視聴回数を最大化する

---

## 1. このプレイブックの考え方

新規チャンネルの1本目では、`広くばら撒く` より `最初の視聴品質を高くする` ほうが重要。

見るべき順はこの5つ。

1. Packaging
2. Publishing
3. Early Distribution
4. Engagement
5. Measurement And Iteration

---

## 2. Packaging

### 2-1. タイトル方針

- `浦島太郎` を先頭に置く
- `AI` は差分として足す
- 余計な説明を足しすぎない
- 釣りではなく、動画内容と期待値を揃える

### 2-2. 推奨タイトル候補

第一候補:

`浦島太郎をAIが語り直す`

予備候補:

- `浦島太郎 | AIで再構築する昔話`
- `昔話「浦島太郎」をAIで映像化`
- `AIでつくる浦島太郎、海と時間の物語`

### 2-3. 説明文の最初の2行

```text
浦島太郎をAIが語り直す、昔話の再解釈です。
竜宮城の幻想、時間のずれ、帰れない切なさを、映画的な空気感でまとめました。
```

### 2-4. hashtag / tag / category

- Hashtags: `#浦島太郎 #生成AI #日本昔話`
- Tags: `浦島太郎, うらしまたろう, Urashima Taro, 日本昔話, 昔話, 生成AI, AI動画, AI映像`
- Category: `Entertainment`

### 2-5. やらないこと

- タイトルにチャンネル名を入れる
- ハッシュタグを盛りすぎる
- 内容にない語をタグへ混ぜる
- description 冒頭をリンクや自己紹介にする

## 3. Publishing

### 3-1. 公開タイミング

第一候補:

- `火曜 19:00 JST`

代替:

- `木曜 19:00 JST`

### 3-2. 公開フロー

`Private -> 最終確認 -> Scheduled Public`

- `Unlisted` は少人数確認だけに使う
- 本番公開の導線としては使わない

### 3-3. 公開30分前チェック

- 冒頭30〜60秒を再視聴
- タイトル先頭に `浦島太郎`
- サムネがスマホで読める
- `Not made for kids`
- コメント保留モデレーション ON
- 説明文先頭2行を確認
- 固定コメント文を準備
- playlist 紐づけ確認
- 公開時刻確認

### 3-4. 公開後30分チェック

- watch page を desktop / mobile で確認
- 固定コメントを pin
- 最初のコメントに返信
- warm network 共有を開始
- Studio で impressions / CTR / AVD を軽く確認
- 明確な誤字や壊れた設定だけ直す
- タイトルやサムネを連打で変えない

---

## 4. Early Distribution

### 4-1. 初動24時間の順番

1. YouTube で公開
2. 0〜1時間で warm DMs / email
3. 0〜2時間で X に native post
4. 2〜6時間で niche community 1〜2箇所
5. 6〜12時間で X follow-up
6. 12〜24時間で必要なら第2波

### 4-2. warm network で送る相手

- AI 制作に関心がある人
- 昔話や物語好き
- 映像制作に関心がある人
- 正直な感想を返してくれる近い人

量より質。
`10〜20人` に絞る。

### 4-3. X 投稿フック

- `「浦島太郎をAIが語り直す」1本目を公開しました。竜宮城の“あの感じ”を、映像で。`
- `知っている浦島太郎と、映像にした浦島太郎は少し違う。`
- `初回は浦島太郎。AIでどこまで物語を持たせられるかの実験。`
- `にわかのAI、1本目を公開しました。制作の裏側も今後出します。`

実務:

- メイン投稿は native image
- YouTube リンクは reply 側に置く

### 4-4. やらないこと

- Premiere を使う
- 無関係コミュニティへ大量投下
- 同じ文面を全部に貼る
- `AIで作りました` だけを主語にする
- `高評価と登録お願いします` を主目的にする

### 4-5. 期待値

- 数十再生: 弱い初動
- `100〜500`: 現実的に狙う帯
- `500+`: 強い初日
- `1,000+`: breakout

## 5. Engagement

### 5-1. pinned comment 方針

- 公開直後に自分で pin する
- 質問は `1つだけ`
- 動画内容に触れる問いにする
- 外部リンクは pin に置かない

### 5-2. 推奨 pin comment

第一候補:

`この動画の「浦島太郎をAIが語り直す」で、いちばん印象に残った場面や、もっと見たかったところがあれば教えてください。短い感想でも、次の作品づくりにかなり助かります。`

予備候補:

- `竜宮城の中で、一番不気味だった / 美しかった場面を1つ教えてください。`
- `もしこの物語を浦島太郎ではなく乙姫側から見るなら、どこが一番気になりますか。`

### 5-3. 説明欄 CTA

コメント CTA は1本に絞る。

```text
一番印象に残った場面、または解釈が気になった点をコメントで教えてください。
```

### 5-4. 返信方針

- 最初の24時間は `Newest` で追う
- 有用コメントには早めに返信
- `ありがとう` だけで終わらせず、言及ポイントを1つ拾う
- 良いコメントには heart

## 6. Measurement And Iteration

### 6-1. 1時間

見るもの:

- impressions
- views
- CTR
- title / thumbnail の表示崩れ
- traffic source mix

判断:

- ここは wiring check
- panic しない

### 6-2. 24時間

見るもの:

- CTR
- average view duration
- audience retention
- Browse / Suggested / Search の比率

基準:

- CTR `4%以上`: 良い
- CTR `2%未満`: 弱い
- retention `35%以上`: 強い
- retention `30%未満`: 弱い

### 6-3. 72時間

見るもの:

- impressions の伸び
- CTR の維持
- retention の維持
- Browse が入っているか

### 6-4. タイトル変更の判断

- `24h` で `CTR <2〜3%`、かつサンプルが十分あるならタイトル変更を検討
- 変更は `1回だけ`
- description は明確に弱い場合だけ直す

### 6-5. 初期判断テーブル

| Window | Signal | Action |
|---|---|---|
| 0–24h | impressions が少ない / retention 未確定 | 待つ |
| 24h | CTR >= 4% かつ retention >= 35% | 触らない |
| 24h | CTR < 2% かつ retention < 30% | タイトルを1回だけ変更検討 |
| 48h | impressions < 500 かつ Browse なし | packaging 修正を1回 |
| 72h | CTR 強いが retention 弱い | タイトル期待値を動画実態へ寄せる |
| 72h | CTR も retention も強い | 何もしない |

## 7. 実行順

### 前日まで

1. タイトル確定
2. 説明文先頭2行確定
3. hashtag / tag / category 確定
4. pin comment 確定
5. warm network リスト作成

### 当日 公開30分前

1. Private で最終確認
2. 公開時刻確認
3. mobile / desktop 確認
4. comment settings 確認

### 公開直後

1. pin comment
2. warm network 共有
3. X 投稿
4. 初回コメント返信

### 24時間以内

1. niche communities へ限定共有
2. X follow-up
3. metrics 確認

### 72時間以内

1. title を触るか判断
2. 次回改善点を記録
