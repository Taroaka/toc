# Design: YouTube サムネイル用プロンプトを返す専用エージェント

## 方針

### A) 役割

新エージェントは `画像生成を実行する役割` ではなく、
`画像生成用の構造化プロンプトを返す役割` に限定する。

これにより、モデルや画像生成基盤に依存せず、
ユーザーは好きな画像生成ツールへ同じ出力を流し込める。

### B) 入力優先順位

入力の優先順位は次の通り。

1. ユーザーが指定した物語名
2. ユーザーが指定した run dir
3. 指定 run dir 内の `story.md` / `visual_value.md` / `video_manifest.md`
4. 該当成果物が無い場合は、物語名から一般的なモチーフを推定

run dir が利用できる場合は、
背景モチーフや色味を成果物に寄せて精度を上げる。

### C) 出力契約

エージェントはユーザーへの返答として、
最低限次の構造で返す。

- `story_title`
- `thumbnail_goal`
- `canvas`
- `text_design`
- `background_direction`
- `composition`
- `quality_constraints`
- `avoid`
- `final_prompt`

`final_prompt` には、以下を必ず統合する。

- 物語名の大きな装飾文字
- 物語イメージから連想される独創的デザイン
- 物語に合った背景
- 文字可読性の担保
- YouTube thumbnail 向けの 16:9、高解像度、視認性、余白

### D) YouTube サムネ向け固定制約

- アスペクト比は `16:9`
- 解像度は `1280x720` 以上を推奨
- 中央または主要視線導線にタイトル文字を大きく配置
- 小画面でも読めるように高コントラストを確保
- 背景情報は盛りすぎず、文字が埋もれない密度に制限
- 実写風 / 映画ポスター風に寄せてもよいが、
  「既成フォント感」は避け、文字自体を造形する

### E) 可変要素

作品ごとに次を変える。

- 文字装飾の素材感
  - 例: 桃、鬼、海、玉手箱、竹、月
- 背景の場所性
  - 例: 山道、浜辺、竜宮城、竹林
- 色設計
  - 例: 桃太郎なら桃色 + 朱 + 墨、浦島太郎なら群青 + 金 + 珊瑚色

### F) 保存ポリシー

- デフォルトではユーザーへの返答のみを行う
- ファイル保存はユーザーが明示した場合のみ行う
